Prompt tips
Direct action verbs saves tokens
Start with the verb. Claude responds better to commands than to polite requests. Skip the preamble entirely.
✕ "Could you help me write an email to my boss about..."
✓ "Write an email to my boss about..."
Say how deep you want it adds tokens
Claude 4.x won't go "above and beyond" unless you explicitly ask. Vague adjectives like "good" or "proper" are ignored.
✕ "Write a good summary"
✓ "Write a summary. Be thorough, cover all key points."
Give Claude permission to say idk saves tokens
Without this, Claude will guess and fill gaps with hallucinated info. One sentence prevents this.
✕ "What is the exact population of Chennai?"
✓ "What is the population of Chennai? If unsure, say so."
Add context / motivation worth it
Explaining why you need something helps Claude make better decisions about related choices automatically.
✕ "Don't use bullet points"
✓ "No bullet points — I need flowing prose for a formal document"
Chain-of-thought for hard tasks adds tokens
Adding "think step by step" makes Claude reason before answering. Costs more tokens in the reply but gets better accuracy on math, logic, code.
✕ Use for: simple factual questions
✓ Use for: debugging, calculations, multi-step plans
Use examples (few-shot) adds tokens
One example is worth more than a paragraph of description. Show Claude the format/style you want instead of describing it.
✕ "Write in a casual but professional tone"
✓ "Match this tone: 'Hey! Quick update on the project...'"
What are XML tags — and when do they cost tokens?
<context> Background info Claude needs before the task. Helps it understand your situation. +4 tok
<instructions> The actual task. Claude treats this as the primary directive. +6 tok
<example> Show Claude the format/style you want. One example outperforms long descriptions. +4 tok
<constraints> Rules Claude must follow — word limit, tone, audience, format. +5 tok
<files> References attached files by name so Claude knows what to look at. +3 tok
<thinking> Tells Claude to reason inside this tag before answering. Used with chain-of-thought. +4 tok
Total cost of full XML structure: ~25–30 tokens. That's roughly the same as one extra sentence of filler text. Worth it for complex prompts — skip it for simple one-liners.

Key insight: On the free plan, a single prompt asking for "a detailed 1000-word essay" might use as many tokens as 5–10 short Q&A messages combined. The reply is usually 3–10× more tokens than the prompt. Always specify reply length if you're watching your budget.
